# Milestone Roadmap

## Milestone 0: product framing
Deliverables:
- product one-pager
- v1 scope/non-goals
- UX direction principles
- terminology glossary
- success criteria

## Milestone 1: desktop foundation
Deliverables:
- Electron shell
- layout/navigation skeleton
- secure local storage integration
- privileged/local boundary implementation
- local workspace persistence model

## Milestone 2: runtime foundation
Deliverables:
- canonical entity model implementation
- connector manifest support
- skill manifest support
- capability resolution
- run history model
- executor abstraction

## Milestone 3: first integrations
Deliverables:
- Jira connector
- Outlook Calendar connector
- Teams or Slack connector
- connector status/freshness UI

## Milestone 4: flagship workflow
Deliverables:
- daily command-center workflow
- calendar view foundation and event detail panel
- generated operational artifacts
- Today view
- basic triage and follow-through surfaces

## Milestone 5: terminal integration hardening
Deliverables:
- embedded terminal experience or strongly integrated terminal pane
- workflow launch actions
- artifact-aware execution context
- improved run logs and approvals

## Milestone 6: product hardening
Deliverables:
- packaging/distribution
- updater pipeline
- crash handling
- redaction-aware telemetry policy
- onboarding and settings polish

## Milestone 7: sync preparation
Deliverables:
- sync-ready local data boundaries
- data classification matrix
- encrypted sync architecture draft
- account/subscription model draft

## Milestone 8: premium Fred preparation
Deliverables:
- entitlement model
- Fred integration boundary
- premium workflow concepts
- AI-safe data minimization rules

## Summary

The roadmap should prove the base product first, then prepare the product for sync and premium orchestration without making either one a prerequisite for early user value.
